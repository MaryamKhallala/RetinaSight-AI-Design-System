Boutons, liaisons & exploitation backend

Cette page décrit chaque bouton de l'application, la manière dont les écrans s'enchaînent, et comment la petite base de test (locale, dans le navigateur) se branche au vrai backend FastAPI / PostgreSQL.

Périmètre du prototype. Le prototype tourne entièrement dans le navigateur. Pour le tester, il embarque une petite base de données locale (lib/db.js, persistée dans localStorage) qui rejoue le pipeline IA et stocke cas + rapports. L'API de cette base est identique aux futurs endpoints du serveur : passer en production = remplacer le corps de chaque méthode par un fetch(). Aucun appareil médical réel n'est connecté.

4. Correspondance base locale → backend réel

Chaque méthode de OctopusDB a une signature alignée sur un endpoint FastAPI. Le passage en production ne change que l'implémentation interne.

Méthode (prototype)Endpoint (production)Table PostgreSQL
listCases()GET /api/casescases
getCase(id)GET /api/cases/:idcases
addCase(payload)POST /api/casescases + objet image (MinIO)
runInference(id)POST /api/cases/:id/inferai_results (service GPU)
saveReport(id, txt)POST /api/cases/:id/reportreports
listReports()GET /api/reportsreports
listUsers()GET /api/usersusers (RBAC Keycloak)
listDatasets()GET /api/datasetsdatasets
listModels()GET /api/modelsmodel_registry (MLflow)
listAudit()GET /api/auditaudit_log

Exemple de bascule (une seule méthode)

// Prototype (local) listCases() { return clone(DB.cases); } // Production — même signature, corps remplacé async listCases() { const r = await fetch('/api/cases', { headers: authHeader() }); return r.json(); }
Le backend réel est livré. Code FastAPI + SQLAlchemy dans backend/ (voir backend/README.md). Démarrer petit : cd backend && pip install -r requirements.txt && uvicorn app.main:app --reload (SQLite auto-amorcé, docs sur /docs). Brancher l'interface dessus, dans la console du navigateur : localStorage.setItem('octopus.api_url','http://localhost:8000/api'); location.reload(). L'app hydrate alors ses données du serveur et écrit (téléversement, signature, tentative) vers l'API ; sans cette clé, elle reste en mode local.

5. Pile backend cible (rappel PRD)

FastAPI · Python 3.12 PostgreSQL 16 MongoDB 7 (métadonnées image) MinIO (objets / images) Redis 7 (cache) PyTorch 2 + ONNX (inférence GPU) Keycloak (OIDC / RBAC) Sentry (monitoring)

Pour démarrer petit : un seul service FastAPI + une base PostgreSQL avec les 6 tables ci-dessus suffisent à reproduire l'API de OctopusDB. L'inférence peut d'abord renvoyer une réponse simulée (comme infer()) avant de brancher le vrai modèle.
